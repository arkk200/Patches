# `.patches` 파일 형식 (File Format)

이 문서는 `puzzles` 폴더 내에 저장되는 `.patches` 파일의 작성 규칙을 설명합니다. `.patches` 파일은 Patches 퍼즐의 크기, 초기 보드판 구성, 그리고 각 패치의 세부 속성을 정의합니다.

## 파일 구조

`.patches` 파일은 크게 세 가지 영역으로 구성됩니다.

1. **그리드 크기 (Grid Size)**
2. **보드판 구성 (Board Layout)**
3. **패치 정의 (Patch Definitions)**

---

### 1. 그리드 크기 (Grid Size)

- **위치:** 파일의 첫 번째 줄
- **형식:** `{가로(Width)}x{세로(Height)}`
- **예시:** `5x5`, `6x6`
- **설명:** 퍼즐 보드판의 전체 크기를 나타냅니다.

### 2. 보드판 구성 (Board Layout)

- **위치:** 두 번째 줄부터 (정의된 세로 높이만큼의 줄)
- **기호 설명:**
  - `.` (마침표): 빈 공간을 의미합니다.
  - `a`, `b`, `c` (알파벳 소문자): 각 패치의 기준(숫자/심볼)이 위치하는 곳의 식별자(ID)입니다.
- **규칙:**
  - 각 줄의 기호 개수는 정의된 가로 크기와 일치해야 합니다.
  - 줄의 총 개수는 정의된 세로 크기와 일치해야 합니다.

### 3. 패치 정의 (Patch Definitions)

- **위치:** 보드판 구성 영역 직후부터 파일 끝까지
- **형식:** `{ID}:{Size}:{Shape}`
- **설명:** 보드판에 나열된 각 알파벳(ID)이 어떤 속성의 패치인지 정의합니다.

| 항목        | 의미                                    | 예시 / 값 종류                                                                                                 |
| :---------- | :-------------------------------------- | :------------------------------------------------------------------------------------------------------------- |
| **`ID`**    | 보드에서 쓰인 해당 패치의 알파벳 식별자 | `a`, `b`, `c` 등                                                                                               |
| **`Size`**  | 해당 패치가 차지하는 타일의 개수(숫자)  | `4`, `8` (숫자가 명시되지 않은 경우 `-` 사용 가능)                                                             |
| **`Shape`** | 패치의 모양 형태 조건 (규칙)            | `wide`: 가로로 넓은 모양<br>`tall`: 세로로 긴 모양<br>`square`: 정사각형 모양<br>`any`: 어떤 모양이든 상관없음 |

---

## 예시 파일 분석

`puzzles/3.patches` 파일의 예시는 다음과 같습니다.

```text
6x6           <-- 1. 그리드 크기 (가로 6칸, 세로 6칸)
.a..b.        <-- 2. 보드판 (총 6줄, 각 줄마다 6개의 문자)
..c...
.....d
e.....
...f..
.g..h.
a:4:tall      <-- 3. 패치 정의 (a는 총 4칸이며, 세로로 긴 모양이어야 함)
b:4:square
c:4:square
d:-:wide      <-- (숫자가 없는 경우 하이픈 `-`으로 표기, 가로로 긴 모양)
e:-:tall
f:4:wide
g:4:wide
h:4:square
```
